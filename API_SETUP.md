# Настройка API для разработки

## Текущая конфигурация

Проект настроен для работы с API тремя способами:

### 1. Использование продакшн API (Текущая настройка)
- API URL: `https://appserver.faithtribe.io`
- Подходит для большинства задач разработки
- Не требует локального API сервера

### 2. Использование локального API сервера
Если у вас есть локальный API сервер:
1. Измените в `.env.development`:
   ```
   NEXT_PUBLIC_API_MAIN_ENDPOINT=http://localhost:8080
   ```
2. Запустите ваш локальный API сервер на порту 8080
3. Перезапустите Next.js: `npm run dev`

### 3. Использование прокси режима
Для работы через configurator.site:
1. Используйте: `npm run proxy` 
2. Это использует `.env.proxy` с настройками:
   - API через прокси: `http://localhost:3000/api`
   - Автоматическая переадресация на продакшн API

## Важные файлы

- `utils/axiosInstance.ts` - базовая конфигурация axios
- `utils/common.ts` - эндпоинты API
- `.env.*` файлы - конфигурация окружений

## Debug режим

В `.env.development` настроены debug пользователи:
- `NEXT_PUBLIC_DEBUG_SHOPIFY_ID` - для покупателя
- `NEXT_PUBLIC_DEBUG_DESIGNER_ID` - для дизайнера

Это позволяет пропустить обычную авторизацию при разработке.
